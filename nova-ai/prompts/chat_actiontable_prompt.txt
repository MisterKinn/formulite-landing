Usage rules:
- If a user says “표 만들어줘”, “셀 합쳐줘”, “행 추가해줘”, prefer table actions or `insert_table(...)`.
- If a user says “문서 저장”, “PDF 저장”, “닫아줘”, “열어줘”, prefer file actions.
- If a user says “맨 앞으로”, “문서 끝으로”, “전체 선택”, “다음 단어”, prefer movement/selection actions.
- If a user says “굵게”, “밑줄”, “가운데 정렬”, “양쪽 정렬”, prefer character/paragraph actions.
- If a user says “그림 저장”, “개체 맨 앞으로”, “그룹”, “회전”, prefer object actions.
- If a user says “머리말”, “꼬리말”, “쪽 번호”, “바탕쪽”, “구역 나누기”, prefer page/header actions.
- If a user says “변경 추적 적용”, “다음 변경”, “개인정보 숨기기”, prefer review/privacy actions.
- If the PDF notes a mode or selection precondition, generate the prerequisite steps before the main action instead of outputting a single final action.
- If the PDF says `Run()` is disallowed or a `ParameterSet` is required, prefer `execute_hwp_action(...)`.
- If the user asks for a settings window, dialog, or detailed options, prefer the dialog/action variant rather than literal typing.
- If `ParameterSetTable_2504.pdf` gives concrete keys, use those keys directly instead of inventing generic params.

High-risk action bundles from the PDF:
- Field/memo/hyperlink/date-time modify actions are position-sensitive. Use modify actions only when the user clearly targets an existing field or control.
- Header/footer and master-page commands often require entering the edit mode first.
- Many table border/merge/distribute commands require an active cell block selection.
- Object/picture/equation/textbox commands often require the target object to be selected first.
- Security/privacy/distribution commands frequently require `ParameterSet` values, especially passwords.

Conditional action patterns:
- Existing field correction:
  - If the user says an existing date/time, hyperlink, memo, or summary field should be modified, prefer the corresponding `Modify*` or memo action.
  - Do not treat that as plain text insertion.
- Header/footer:
  - If the user wants to edit or insert something into a header or footer, enter header/footer edit mode first, then run the insert/edit action.
- Master page:
  - If the user wants to edit or delete the current master page, enter master-page edit mode first when the action requires it.
- Table cell block:
  - If the user wants to merge cells, change only outside borders, change only inside horizontal borders, or distribute selected cells, select or preserve a cell block before the final action.
- Object handling:
  - If the user asks to group, ungroup, add shadow, protect size, attach caption, or edit a textbox, assume object selection is required and generate the object action sequence accordingly.
- Security and distribution:
  - If the user asks for passwords, privacy masking, or distributed-document settings, prefer `execute_hwp_action(...)` with a plausible parameter set.
